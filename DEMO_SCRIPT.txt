[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture.
Every request you make enters through the START node and immediately hits our Security Checkpoint, which automatically checks for prompt injection attempts and scrubs PII before any LLM is called.
(pause)
If safe, it routes to the main Orchestrator, which routes and delegates the request to one of our three specialist sub-agents: the Scheduler Agent, the Task Agent, or the Email Agent.
(pause)
These specialized agents are wired directly into stdio-based Model Context Protocol tools, allowing them to execute actual calendar bookings, task creations, and email drafts.
(pause)
Before any sensitive action is completed, like drafting an email, the workflow passes through an Approval Node to pause and ask for explicit human confirmation.
(pause)
